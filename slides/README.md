# Slide deck — 50th-anniversary talk

A ~10-minute Marp slide deck that turns the BibTeX corpus into a
narrative for the **50th Annual Graham Goddard In-House Conference**
(2026). This README is the maintenance manual: how it's built, what each
piece is for, and what to touch when the data changes.

## Build

```bash
cd ~/dev/inhouse-conference-archive
python3 slides/analyze.py        # ~3s: rebuilds figures + stats.json
cd slides
npx --yes @marp-team/marp-cli@latest --allow-local-files --html slides.md -o slides.pdf
```

The Homebrew `marp` binary is broken on Node v26+; **always use `npx
@marp-team/marp-cli@latest`**. The `--allow-local-files` flag is required
because the deck embeds PNGs from the `figures/` directory.

## What's generated vs. hand-curated

**Generated (re-run `analyze.py`):**
- All five PNG figures.
- `stats.json` — full lists of top authors, yearly top, streaks, shortest
  / longest titles, every regex-matched amusing title, every year's
  most-over-represented word.

**Hand-curated in `slides.md`:**
- The **amusing-titles slides** (22, 23). `analyze.py` finds ~360 regex
  candidates; I picked ~8 that read well aloud and aren't OCR-garbled.
  Pool to re-pick from: `stats.json["amusing_titles"]`.
- The **word-of-the-year slide** (24). I picked years for narrative arc
  (kindling → attention → Alzheimer's → trauma); the full per-year list is
  in `stats.json["yearly_word"]`.
- The **corpus-numbers slide** (13). Re-paste from the QA report after
  re-running the pipeline.
- The **conference-name table** (14). Verbatim from
  `pipeline/booktitles.py`; update when the conference is renamed.

## When the data changes — refresh checklist

For a *new conference year* (e.g. adding 2027):

1. Drop `In-House Program 2027.{pdf,...}` into `source_programs/`.
2. Add a `parse_2027` (or reuse an existing parser) in `pipeline/02_parse.py`
   and wire it into the dispatcher.
3. Add a 2027 entry to `pipeline/booktitles.py`.
4. Run `bash pipeline/run_pipeline.sh`.
5. Run `python3 slides/analyze.py`.
6. Update in `slides/slides.md`:
   - Slide 13 corpus numbers ← `qa_report.md` summary.
   - Slide 14 name table only if the name changed.
   - "1975–2026" references throughout → "1975–2027".
   - Spot-check slides 22–24 — new year may have great titles or a
     new dominant word.
7. Re-render to PDF.

For an *abstract correction* (existing year):

1. Edit `corrections.jsonl` or the diff-review loop (see CLAUDE.md).
2. Run `python3 pipeline/03_export_bib.py` (re-applies corrections).
3. Run `python3 slides/analyze.py` to refresh figures/stats.
4. Re-render.

## Style conventions

- **Theme**: Marp default with a custom `style:` block in the frontmatter.
- **Font**: Public Sans (loaded via Google Fonts `@import` in CSS).
- **Title + closing slides**: per-slide directives, required for the
  yellow background to apply (a `section.title { background: ... }` CSS
  rule alone does **not** override Marp's theme background; ask me how I
  know):

  ```
  <!-- _class: title -->
  <!-- _backgroundColor: "#FFD400" -->
  <!-- _color: "#000" -->
  <!-- _paginate: false -->
  ```

- **Colours**: `#FFC72C` accent rule under H1s, `#293F88` for H2 + card
  borders, `#A33E3E` for emphasis (`**strong**`). Dal black/gold reads
  immediately to a Dal audience.
- **Image placement**: `![h:520 bg right:42% contain](figures/foo.png)`
  is the standard "photo on right, prose on left" pattern.
- **Cards**: `<div class="quote-card">…</div>` is the standard wrapper
  for individual titles/quotes; the `.year` and `.authors` spans inside
  pick up the styling defined in the CSS block.

## `analyze.py` — design notes

- Loads `../inhouse_conference.bib` directly (not `records.jsonl`) so
  corrections are already applied.
- `normalise_author` is fairly aggressive: strips OCR garble, splits on
  `;` (rescues fused-author display strings), and flips the `INITIALS,
  Lastname` shape (saved Natalie O. Rosen from being rendered "NO, Rosen"
  in the 2017 top-of-year chart).
- The **shortest-titles** filter caps word count at 5 and skips < 2 words
  to drop truncated stubs.
- The **longest-titles** filter caps at 40 words; anything beyond that is
  almost always a parser concatenation (the all-time record was 276 words,
  which was 8 records fused into one).
- The **co-authorship network** keeps only the largest connected
  component, draws ~108 authors with surname-only labels, font size scaled
  by total presentations. Thresholds (`NODE_MIN_PRES=6`, `EDGE_MIN=2`) are
  tuned for readability — drop them to see more of the long tail.
- The **amusing-titles regex** is a curated list of patterns (animals,
  drugs, body parts, self-deprecation, double-question-marks). Add to
  `AMUSING_PATTERNS` if you find a new genre.

## Known fragilities

- The deck embeds PNGs by relative path. Don't rename `figures/` without
  fixing the markdown.
- The Public Sans `@import` requires internet access at render time.
  Vendored copies live nowhere; if rendering offline, change the font to
  any system sans-serif.
- 2026 is currently the last format dialect in `02_parse.py`. Adding 2027
  with the same format → just extend the year range; with a different
  format → write a new `parse_2027` and dispatch it.
